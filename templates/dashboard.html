Cost & Scalability

Primary Cost Drivers

  • App Service (web): instance size/count; AlwaysOn; egress.
  • Azure Functions (SRE/Info): executions, GB-s, durable storage.
  • Azure OpenAI: tokens per request & model choice.
  • Azure Table Storage: transactions + data-at-rest.
  • Log Analytics: GB ingested & retention.
  • Data Factory: pipeline activity/monitoring overhead.
Keep costs predictable
  • Dedupe by alertId; throttle bursts.
  • Cache AOAI results by error signature hash.
  • Batch Table writes; store links not large blobs.
  • Use Functions consumption for bursty load.
  • Right-size Log Analytics retention/verbosity.

Scalability Patterns

  • Web App: autoscale on CPU/req; tune gunicorn workers/threads.
  • Durable Functions: control concurrency (host.json), fan-out checks.
  • AOAI: respect rate limits; queue & retry with jitter; smaller model for classify.
  • Storage: partition by date/pipeline to avoid hot keys.
  • Resilience: idempotency by alertId/runId; DLQ failures.
  • Observability: webhook & AOAI latency, durable backlog, retry success, MTTR.
Suggested knobs
  • Gunicorn: WEB_CONCURRENCY, THREADS.
  • Durable: maxConcurrentActivityFunctions, storage provider settings.
  • AOAI: region-available model with lower cost for classification.
